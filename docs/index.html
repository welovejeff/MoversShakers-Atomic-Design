Documentation

Complete guide to using the Movers+Shakers Design System in your projects.

Installation

Include the following in your HTML file:

<!-- Tailwind CSS -->
<script src="https://cdn.tailwindcss.com"></script>

<!-- Google Fonts -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&family=Oswald:wght@400;500;600;700&display=swap" rel="stylesheet">

<!-- Lucide Icons -->
<script src="https://unpkg.com/lucide@latest"></script>

<!-- Design System CSS -->
<link rel="stylesheet" href="tokens/index.css">

<!-- Design System JS -->
<script src="js/components.js"></script>

Basic Usage

Components are initialized automatically. Access them via window.MSDesignSystem:

// Show a toast notification
MSDesignSystem.Toast.success('Operation completed!');

// Toggle dark mode
MSDesignSystem.DarkMode.toggle();

// Check if dark mode is active
if (MSDesignSystem.DarkMode.isDark()) {
    console.log('Dark mode is on');
}

React + TypeScript

Complete React component library with TypeScript support.

Installation

# Install from packages/react
npm install @movers-shakers/react

# Or link locally during development
cd packages/react && npm link
npm link @movers-shakers/react

Quick Start

import { Button, Badge, Card, Modal } from '@movers-shakers/react';
import { useTheme, useToast } from '@movers-shakers/react';

function App() {
  const { toast, ToastContainer } = useToast();
  const { theme, toggle } = useTheme();

  return (
    <>
      <Button variant="primary" onClick={() => toast.success('Saved!')}>
        Save Changes
      </Button>
      <ToastContainer />
    </>
  );
}

Web Components

Framework-agnostic custom elements using Lit.

Installation

<!-- Include in HTML -->

Available Elements

Element Usage

Modal

Create modal dialogs with automatic focus management.

HTML Structure

<!-- Trigger Button -->
<button data-modal-trigger="my-modal">Open Modal</button>

<!-- Modal -->
<div data-modal="my-modal" class="hidden fixed inset-0 z-50 items-center justify-center bg-black/50">
    <div class="bg-white border-2 border-black p-6">
        <h3>Modal Title</h3>
        <p>Modal content here...</p>
        <button data-modal-close>Close</button>
    </div>
</div>

Data Attributes

Attribute Description
data-modal-trigger="id" Opens the modal with matching ID
data-modal="id" Defines the modal container
data-modal-close Closes the active modal

Toast Notifications

Programmatic toast notifications.

// Simple notifications
MSDesignSystem.Toast.info('Information message');
MSDesignSystem.Toast.success('Operation completed!');
MSDesignSystem.Toast.warning('Please check your input');
MSDesignSystem.Toast.error('Something went wrong');

// With options
MSDesignSystem.Toast.show({
    message: 'Item deleted',
    type: 'success',
    duration: 5000,
    dismissible: true,
    action: {
        label: 'Undo',
        onClick: () => console.log('Undo clicked')
    }
});

Form Validation

Real-time form validation with built-in rules.

<form data-validate>
    <div data-field>
        <input data-rules="required|email"
               data-error-required="Email is required"
               data-error-email="Enter a valid email">
    </div>
    <div data-field>
        <input type="password" id="password"
               data-rules="required|minLength:8">
    </div>
    <div data-field>
        <input data-rules="required|match:password"
               data-error-match="Passwords must match">
    </div>
</form>

Available Rules

Rule Description
required Field must have a value
email Valid email format
minLength:n Minimum n characters
maxLength:n Maximum n characters
pattern:regex Match regex pattern
match:fieldId Match another field

Dark Mode

Automatic dark mode with system preference detection and localStorage persistence.

Toggle Button

<button data-theme-toggle>
    <i data-lucide="sun" data-theme-icon="light"></i>
    <i data-lucide="moon" data-theme-icon="dark" class="hidden"></i>
</button>

JavaScript API

// Toggle dark mode
MSDesignSystem.DarkMode.toggle();

// Set specific theme
MSDesignSystem.DarkMode.setTheme('dark');
MSDesignSystem.DarkMode.setTheme('light');

// Check current theme
MSDesignSystem.DarkMode.getTheme(); // 'light' or 'dark'
MSDesignSystem.DarkMode.isDark(); // true or false

// Listen for theme changes
document.addEventListener('theme:changed', (e) => {
    console.log('Theme changed to:', e.detail.theme);
});

CSS Variables for Theming

/* Use theme-aware variables */
.my-component {
    background: var(--color-bg-primary);
    color: var(--color-text-primary);
    border-color: var(--color-border-primary);
}

/* Or use utility classes */
.my-component {
    @apply bg-theme-primary text-theme-primary border-theme-primary;
}
